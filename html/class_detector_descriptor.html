Detailed Description

Base class manages the detection and description of keypoints.

This class allows the extraction of a set of features in an image, the computation of descriptors and display of keypoints. The implemented dectectors find interesting pixels from an image to allow an efficient matching with other images. Detectors and descriptors are implemented with default values. The detectors AGAST, FAST and HARRIS do not include descriptors. Therefore they should be associated with other descriptors. The descriptors KAZE and AKAZE should be used only associated with themselves as detectors.

distributeKeypoints : Detection and distribution of the keypoints

Computes the keypoints of the image stored with the detector specified in the constructor. The resulting keypoints are stored in the keypoints list of a featuresList. All keypoints are distributed equally in boxes for an homogeneous repartition of keypoints and a better homography computation m_thresh should be between 2 and 30 (according to BRISK value)

Parameters
[in]min_thresh: if a kpts response is higher the kpts is selected until the box has as many kpts as required. If there is no such kpts enough, the program fill the image zone with kpts with response below
[in]keypts_desired: number of keypoints desired in the whole image
[in]nb_x: number of boxes horizontally
[in]nb_y: number of boxes vertically
Returns
-1 if the type of detector does not exist 0 otherwise.
