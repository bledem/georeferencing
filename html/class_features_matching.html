Detailed Description

Base class which manages the matching of keypoints from 2 images.

This class allows the matching of descriptors, the storage of corresponding keypoints in a external text file and the display of matched keypoints. The attributes of the class are instanciated empty and are filled by the methods.

FeaturesMatching::FeaturesMatching ( std::string  _method)

Constructor.

Constructor of the class FeaturesMatching

Parameters
[in]_method: type of the matcher. The supported methods are :
  • BruteForce : it uses the L2-norm. It is the preferable choice for SIFT and SURF
  • Brute Force-L1 : it uses the L1-norm. It is recommended for KAZE
  • BruteForce-Hamming : uses the Hamming distance which is recommended for ORB, BRISK and AKAZE
  • BruteForce-Hamming(2) : is recommend with ORB when WTA-K is 3 or 4, which correspond to the number of points that produce each bin of the descriptor. By default WTA-K is 2 in the implemented ORB.
Returns
Nothing.

FeaturesMatching::~FeaturesMatching ( )

Matching of keypoints.

Matchs keypoints from two images. In this method the descriptors are compared one by one and the distance is computed. The Lowe ratio is used to filter ambiguous matches.

Parameters
[in]features1: the featureSet of the first image
[in]features2: the featureSet of the second image
[in]maxDistance: threshold for the distance between matched descriptors. Distance means here metric distance (e.g. Hamming distance), not the distance between coordinates (which is measured in Pixels)
Returns
-1 if no matcher is provided 0 otherwise

std::vector<cv::DMatch> FeaturesMatching::matches

List of matches. The DMatch structure contains:

  • int queryIdx : query descriptor index , ie the index of the descriptor to retrieve in any other image
  • int trainIdx : train descriptor index , ie the index of the corresponding descriptor in the train image
  • int imgIdx : the train image index
  • float distance : the distance between descriptors
